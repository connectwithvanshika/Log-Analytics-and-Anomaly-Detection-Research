# Observability Research Lab

A research-driven repository documenting my journey into **log analytics, anomaly detection, distributed tracing, and root cause analysis**, inspired by modern observability platforms such as Dynatrace.

The primary goal of this project is to understand how enterprise observability systems detect anomalies, correlate events, identify root causes, and provide actionable insights across large-scale distributed environments.

---

## Research Motivation

Modern applications generate massive amounts of telemetry data in the form of:

* Logs
* Metrics
* Traces
* Events
* Infrastructure Data
* User Experience Data

Platforms such as Dynatrace transform this raw data into meaningful insights using topology mapping, machine learning, causal AI, and graph-based analysis.

This repository serves as a research notebook where I document my findings while studying these systems and exploring how similar capabilities can be built from scratch.

---


## Research Questions

Some of the questions driving this research include:

* How do observability platforms collect telemetry data at scale?
* How are logs, metrics, and traces connected together?
* How do systems learn normal behavior without manually defined thresholds?
* Which machine learning algorithms are suitable for anomaly detection?
* How are graph algorithms used for event correlation?
* How is root cause analysis performed automatically?
* Can a lightweight Dynatrace-inspired system be built using open-source tools?

while understanding the engineering principles used by industry-leading platforms such as Dynatrace.
